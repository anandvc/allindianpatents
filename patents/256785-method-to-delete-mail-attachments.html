Title of Invention

METHOD TO DELETE MAIL ATTACHMENTS
Abstract The invention proposes a method and system for deleting attachments in emails present in the mailbox of the MFP to free up memory space of the mailbox. According to the method, the contents of the emails are retained, while the attachments can be marked by a user for deletion. Further, the user can also be provided with an option of automatic deletion of emails based on certain conditions like age of the email, getting a new email containing the latest version of the attachment, etc.
Full Text FIELD OF THE INVENTION
The present invention, in general, relates to the field of embedded system and particularly to multi-functional peripheral (MFP). More particularly, the present invention relates to a method and system to delete mail attachments.
DESCRIPTION OF RELATED ART
The patent publication GB2377135A describes an email manager program for automatic deletion of attachments of emails using predefined rules. The predefined rules can include age of the email, the size of the email, etc. Further, the predefined rules can be customized by the user. However, the publication describes the email manager for wireless information devices like smart phones, communicators, etc.
The patent publication WO2001084764A2 discloses a concept of dynamically removing attachments from received messages. However, the publication focuses on handheld computers and cell phones to describe the concept, rather than on an MFP. Moreover, the publication mentions about filtering of text and attachments, rather than permanent deletion of attachments from a mailbox.
The concept of deleting attachments of received emails from the mailbox, while retaining the contents of the emails, has been disclosed in the patent publication GB2377135A. The publication also discloses the concept of allowing a user to set

conditions for automatic deletion of attachments based on age/size of the attachments. However, the publication specifically describes the concept for wireless information devices.
The patent publication WO2001084764A2 also discloses a concept of setting policies for receiving messages on handheld devices. However, the attachments are deleted from the messages before the messages are received on the devices.
SUMMARY OF THE INVENTION
MFPs have mail box which stores the Mail. The Mail sometimes has attachments which uses a lot of memory. The attachments become obsolete after sometime. For E.g. the attachment of a document which has a different version of document (latest) in some other mail. In this case the older document becomes obsolete. Hence it can be deleted but the Mail contents needs to be maintained.
The main object of the invention is to provide a method for deleting attachments in emails present in the mailbox of the MFP to free up memory space of the mailbox. Another object of the invention is to provide a system for deleting attachments in emails present in the mailbox of the MFP to free up memory space of the mailbox.

Accordingly, this invention explains a method for deleting attachments in emails present in the mailbox of the MFP to free up memory space of the mailbox comprising the steps of:
(a) marking the attachments by a user for deletion;
(b) retaining the contents of the email to be deleted; and
(c) removing the marked attachment.
Accordingly, this invention further explains a method for deleting attachments in emails present in the mailbox of the MFP to free up memory space of the mailbox comprising the steps of:
(a) providing an option of automatic deletion of emails based on user set conditions;
(b) scanning the mailbox for attachments to be removed on the set conditions when a new mail arrives and the mail box is running out of memory space;
where the contents of the email to be deleted is retained.
The user set conditions involves age of the email and getting a new email containing the latest version of the attachment.

The present invention relates to the field of Multi Functional Peripherals (MFPs). More specifically, the invention proposes a method for deleting attachments in emails present in the mailbox of the MFP to free up memory space of the mailbox. According to the method, the contents of the emails are retained, while the attachments can be marked by a user for deletion. Further, the user can also be provided with an option of automatic deletion of emails based on certain conditions like age of the email, getting a new email containing the latest version of the attachment, etc.
MFPs have mail box which stores the Mail. The Mail sometimes has attachments which uses a lot of memory. The attachments become obsolete after sometime. For E.g. the attachment of a document which has a different version of document(latest) in some other mail. In this case the older document becomes obsolete. Hence it can be deleted but the Mail contents needs to be maintained.
MFPs have Mail Box that can store Mails and Mails with attachments. If the user feels the attachment is obsolete then he can mark for deletion of the attachment. In this case attachment is removed and space occupied by the attached is

released to free memory. Also the user can have the options set as -"automatically remove obsolete documents".
When a new mail comes and the Mail Box is running out of space then the mailbox will be scanned for obsolete documents, and the older ones can be removed. Also the user has an option only to delete the attachments and maintain the Mail contents.
The user can configure the MFP to do the above in a customized way.
One of the ways to remove obsolete attached documents can be to check for same name of attachment and delete older one. The user can configure that this can be applied only for some particular address, or for a particular group, etc. As the conditions for obsolete attachments varies from one user to other user this is user customized.
