METHOD AND APPARATUS FOR SELECTING ITEMS FROM A COLLECTION OF ITEMS
Abstract Techniques are presented for selecting one or more items from a collection of items. To select the one or more items, an interface is provided that is adapted to allow a user to define one or more weights. Each weight corresponds to one of a number of similarity criteria. Each item also corresponds to the number of similarity criteria. The one or more weights define a similarity function. The similarity function is applied to the one or more similarity criteria corresponding to the one or more weights and to each of the items in order to select one or more items from the collection of items. The interface can comprise movable markers corresponding to similarity criteria. Locations of the movable markers can be used to weight similarity criteria when creating the similarity function.

The present invention relates to computer processing of information, and more particularly, to computer processing of hems such as artists, songs, movies, pictures, television shows, and news stories.
When using a computer system, a user may desire to select items from a number of hems. For instance, a user might wish to select a playlist of songs from his or her repository of songs. The selected songs could then be played.
When the repository is small, selecting a playlist is quite easy, as there are not that many choices. As the number of songs in the repository grows, however, selecting a playlist can become problematic. For instance, for someone having hundreds or even thousands of songs, h can be hard to remember which songs from which artists are available. Additionally, the process of selecting an artist then selecting a song by that artist can be laborious when a user wants a playlist from multiple artists, especially if the artists and songs are stared in directory structures that have to be perused.
New multimedia programs have helped this process. As an example, certain new multimedia programs can categorize hems into media libraries that can then be searched or organized. Additionally, multiple playlists can be saved in the media libraries.
Nevertheless, there are instances when it would be beneficial for a computer system to select hems for a user. As an example, when a user is playing a song and would like to hear similar songs, h would be beneficial for a computer system to determine and display the titles of similar songs for the user. A need therefore exists for improved methods and apparatus for selecting hems, such as digitally stored songs, from a list of hems, so that, for instance, hems similar to a pre-selected hem may be presented to a user.
Furthermore, a user typically uses an interface when determining what songs to select When a computer system is going to select songs for the user, the user would like to have some control over what songs are selected. It would be beneficial if the controls for such an interface are simple and relatively easy to understand, yet provide a degree of control over how similar songs are determined.
Generally, methods and apparatus are disclosed that select hems from a number of hems.

In an exemplary aspect of the invention, techniques are presented for selecting one or more items from a collection of hems. To select the one or more items, an interface is provided that is adapted to allow a user to define one or more weights. Each weight corresponds to one of a number of similarity criteria. Each hem also corresponds to the number of similarity criteria. The one or more weights define a similarity function. The similarity function is applied to fee one or more similarity criteria corresponding to the one or more weights and to each of the set of hems in order to select the one or more hems from fee collection of hems.
Thus, a user can define a similarity function, which can be used to select items of interest to the user.
In another aspect of fee invention, a user selects a predetermined item of the collection of hems. In order to determine hems deemed similar to fee predetermined item, a similarity value is determined for a given hem. The similarity value represents a similarity of fee given hem wife fee predetermined item. The determination of fee similarity value uses values for each of one or more similarity criteria corresponding to both the given hem and fee predetermined hem. The step of determining a similarity value is repeated until similarity values for each of fee plurality of hems other than fee predetermined hem have been determined. This allows a user to be, for instance, presented wife hems similar to fee predetermined item.
A weight may also be determined for each of a number of similarity criteria. The determination of fee similarity value can use a combination of weighted values for each of fee similarity criteria corresponding to both fee given hem and fee predetermined hem. The weighted values are determined using the weights and fee values of fee one or more similarity criteria.
Similarity values may be determined by histograms of similarity criteria, where fee histograms correspond to artists, for instance, such as song writers, television actors, or movie producers.
The hems can be, for instance, artists, songs, movies, pictures, television shows, and news stories. Items can be selected based on corresponding values of fee similarity values and fee selected hems can be presented to a user. The similarity criteria can be, for instance, item names (e.g., song names), artist names, activity periods, tempos (e.&, of songs or movies), distributions of song features, beat strengths, genre names, style names,

sounds, moods, topics, and director names.
In another exemplary aspect of the invention, an interface region comprises two or more dimensions and is displayed to a user. Markers are located on the interface region and correspond to similarity criteria. Locations on the interface region are determined for the markers. A weight is determined for each of the markers based on a corresponding location of a given marker. Additionally, each weight can be determined based on a distance between the given marker and a predetermined location of the interface region. The determined weights can be used when determining the similarity values.
In another exemplary aspect of the invention, the markers on the interface region are used to modify weights for the similarity criteria to automatically generate a list of songs or other hems. The weight for each criterion can be determined by computing the distance between the given marker and a predetermined location of the interface region. The weights of the similarity criteria can determine the number of hems, in the automatically generated list, desired by the user for those specific similarity criteria. Thus, the user can have a high degree of control over hems in the list, without ever selecting a predetermined hem.

As described above, there are problems with selecting items from a number, e.g., collection, of hems. For example, consider the task of manually compiling a playlist of songs. With conventional navigation software, the user selects first the artist, then the album, and eventually the song desired- The process is repeated for each song the user wants to include in the playlist This does not seem problematic, as it can be clear to the user what music should be in the playlist However, some people can download so many songs that they arc not sure what songs are in their collections. Moreover, even if a user knows all the music in his or her collection, it can be laborious to create playlists, especially if the user wants to vary songs in playlists frequently.
In general, playlists are not arbitrary collections of music. Instead, the music is to some extent coherent with some parameters mentally chosen by the user. Exemplary embodiments of the present invention describe methods and apparatus to help the user build a coherent playlist by presenting to the user relevant information (e.g., similar artists or similar songs) based on an item (e.g., a selected song) selected by the user.
