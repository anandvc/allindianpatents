Title of Invention

A METHOD FOR POWER RESERVATION IN MOBILE DEVICES
Abstract This invention relates in general to the field of mobile and other handheld devices. Further, this invention is applicable to power management in mobile handsets, smart phones and other similar handheld devices. More particularly, this invention relates to a system and method for power reservation in mobile and other handheld devices. This invention explains a method and system for power reservation in a mobile device comprising the steps of: giving the input for power reservation by the user; calculating the required power to reserve by the mobile device with regard to the given inputs; reserving the calculated power based on the availability by the mobile device; providing guidance to the user to facilitate the power reservation by the mobile device; optionally allowing the user to turn off other power rich features by the mobile device; alerting the user of reserved power by the mobile device; and taking a decision on the reserved power by the user as per the alert from the mobile device.

DESCRIPTION OF RELATED ART
There is always a high need for on demand power for the mobile users who travel to the locations where battery charging facility is not available.
When the low battery warning comes, after that, power rich features like Camera, Music player, FM Radio etc., will not be started due to power constraint.
There is no way currently to identify when the low battery will occur, so that before hand we can use power rich features.
Currently user gets the battery-low indication when the battery's remaining power goes below the threshold value set by the manufacturer. Even though Ul Bars (battery annunciators) indicate the power remaining, they are not reliable.

Even though currently there is no way to reserve the battery power for future use, as a part of efficient power management process, 'Power save' mode is employed wherein the mobile enters the low power profile by temporarily turning off LCD, keypad backlights.
There is no way currently to identify when the low battery will occur, so that before hand we can use power rich features.
'Power save' mode as its name implies, it is only for preserving the power, not to reserve it. So user always has to have an eye on the battery level. If the user does not monitor the battery level, occasional moments may be missed from being captured.
SUMMARY OF THE INVENTION
Even though today's available mobiles provide a way to preserve the battery power by 'Power Save Mode', they don't have the facility to reserve the battery power for future use.
This invention provides the user with the ability to reserve the battery power for his/her future use. This system will ensure that the user does not miss his/her occasional moments because of sudden unavailability of battery power.

STRUCTURE OF THE INVENTION
This invention introduces the concept of calculation of power need for a device to

run for a specified length of duration.
Here device refers to power rich features in a mobile like Camera, Music Player, FM Radio etc.
To calculate the device run power, first the low battery threshold value is taken as the device start power as after the low battery warning came, power rich features cannot be started.
After that, device settings power for a second, with a particular profile is calculated. For the length of the length to use, this device settings power is multiplied by X seconds.
Then the final device run power is arrived by adding the device start power with device settings power for the specified length of the period.
Also this invention introduces the concept of 'Device Power Profile Calibration' to determine the power need of a device profile.
With default setting, these device profiles will have the factory set values.
But whenever user updates any of these profiles by modifying the attributes values, then the mobile will perform self power calibration for the updated profile by the Device Profile Power Calibration Procedure.

Also this invention introduces the concept of calculating the available power for reservation in terms of seconds with the current device profile. When the user selects a different profile of a device, mobile automatically calculates the available power for reservation in terms of seconds by dividing the available power by power need of the current device profile for 1 second.

GLOSSARY OF TERMS AND THEIR DEFINITIONS
Annunciator: A small pictorial indicator of the status of some feature
Device: A power rich feature of a mobile like Camera, FM Radio etc.,
Device Profile: Settings of a device attributes
Device Start Power: Minimum battery power required to start the device
Device Run Power: Minimum battery power required to run the device for the specified length of duration
Power Calibration: A procedure to find out the power need of a particular feature for the specified length of duration
Threshold power: Minimum battery power required to use some feature (device) for a specific duration
