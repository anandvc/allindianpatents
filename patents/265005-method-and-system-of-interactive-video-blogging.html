Title of Invention

METHOD AND SYSTEM OF INTERACTIVE VIDEO BLOGGING
Abstract The present invention relates to a method, which helps users to create and participate in Video Blogging by inserting comments in the form of text, audio & audio-video instantaneously at suitable location while watching original blog in Real Time without modifying it. The comments inserted by user become a separate descriptor file & user needs to upload only descriptor file & not the video blog. Every comment uploaded by any user is stored as a separate descriptor file at the server. If there are multiple comments available on one blog then user can select specific comments as per his wish for downloading & watching.
Full Text FIELD OF THE INVENTION
The present invention in general relates to the field of multimedia applications. The present invention involves to the use of Mobile phone with camera facility: Both Feature Phone (Mobile Phone) and Smart phone etc., handheld devices with camera, desktop application, network connectivity etc, and Desktop devices, PC. More particularly, this invention relates to a method of Interactive Video Blogging.
DESCRIPTION OF RELATED PRIOR ART
Patent publication number (W0/2004/102855) , titled "Content Publishing Over Mobile Networks" explains a system for using mobile phones for the purpose of generating instant and permanent publishing of text, images and audio files as so-called "mBlogs" over mobile networks. The system allows the user to generate and publish text, attach image files and audio files with time and location of the event as a non-revocable and integral part of the published content. Users are allowed to view and interact with the published content with mobile phones over mobile networks. The system allows for sorting of content by category and by indexing the material by the operator of a mobile network and allows the users of mBlogs to search for content by category as well as by the
X. means of indexing. Furthermore the system allows users to subscribe to mBlogs as multimedia messages for viewing on mobile phones over mobile networks.
LIMITATIONS
• The current approach allows user to create blog from mobile/PC with still images & text. Also other users can participate in blogging by adding text comments or still images. The blogging does not allow adding information in the form of audio-video as a blog or adding audio video comment or combination of text, audio, video comment from mobile/PC.
• The current approach of video blogging is inefficient & does not allow user to express his views in interactive way as it lack methods to co- relate user comment & original content.
• Entering text as a comment is difficult & cumbersome particularly if user is using handheld device like Mobile phone. It consumes lot of user time. It is difficult to associate text comment with specific portion of video blog. Allowing user to insert comment in the form of audio video adds more value & meaning which is not possible with the current approach.
• The information or comments added by other users in video blog is just one after another & there is no mechanism currently available to co- relate user comments & specific portion of the original video blog. User may wish to add his audio, video or text comment with respect to specific point in original video while watching. But currently there is no method available that allows user to create audio, video or text comment with respect to a specific portion of the video or inserting comment at specific time while watching.
• Original blog & comments added by other participants are inseparable. All comments are put just one after another & every comment or information added by any user becomes integral part of the blog due to which the blog grows in size depending on the number of users participated & the information added by each user.
• User has no provision to see the comments added by various users with respect to specific portion or instant of the original blog. He cannot see the added comments interleaved with original blog with respect to time and content, keeping the original blog and comments as separate entities.
• User needs to download/stream complete blog having original content & comments added by all users every time whenever he wants to watch or participate.
• As video blog size grows depending on the number of users participated & the content added by them, user who participate late feels a burden of uploading & downloading/streaming video blog. It overloads the server, network & consumes user time & money.
• User has no choice of selecting specific comments of his interest made on his favorite video blog. He needs to download/stream & watch complete blog in order to see who has commented on the blog. He needs to download/stream & watch complete blog in order to see just one comment added by his friend.
• User has no choice to select particular comments & watch/play directly.
• Currently there is no mechanism or method available which allows user to insert comment (text, audio, audio-video) on original blog & upload only the comment. He needs to upload original blog, all previous comments & new comment inserted by him.
• Currently there is no mechanism or method available which allows user to insert comments (text, audio, audio-video) on original blog instantaneously & upload only comments associated with specific
portion, specific instant of the original video blog.
• Currently there is no mechanism or method available, which allows user to see original blog & comments (text, audio, audio-video) inserted till now by various user separately (each comment is a separate entity).
• Currently there is no mechanism or method available which allows user to see original blog & all comments (text, audio, audio-video) on original blog separately associated with specific portion, specific instant of the original video blog.
• Currently there is no mechanism or method available which allows user to see original blog & all comments (text, audio, audio-video) on original blog, comment to comment (comment of one user associated with another user's comment) separately associated with specific portion, specific instant of the original video blog or comment of original video blog.
• Currently there is no method which allows user to see original blog, each comment (text, audio, audio-video) added by other users as a separate entity & therefore cannot download/stream original blog along with comments of his choice.
• User may wish to insert comment (text, audio, audio-video) at a specific time while watching original blog & may want other users to see his inserted comment played at right/same time while watching video blog. The insertion of comment while watching blog by one user & play-out of original blog along with inserted comment at right time is not supported by current approach.
• User may wish to insert multiple comments in various forms (text, audio, audio-video) at different point of time while watching original blog & may want other users to see his inserted comments played at right/same time while watching video blog. The insertion of comment in various forms while watching blog by one user at various point of time & play-out of original blog along with inserted comments at right time is not supported by current approach.
• User may have a video blog along with comments made by various users in various forms like text, audio, video. User has no choice of selecting comments category wise like text, audio, audio-video etc.
• User has no mechanism to play the comments added by his choice of friends. User has no mechanism to randomly select the comments he wishes to see.
• User may have a video blog along with comments made by various users in various forms like text, audio, video. User cannot select a particular comment & play/watch directly the comment etc
• User may have a video blog along with comments made by various users in various forms like text, audio, video. User can not select a particular comment & play/watch directly the comment along with associated video or specific portion of the video etc
The present invention has been designed to address these needs.
SUMMARY OF THE INVENTION
The present invention relates to a method for Video Blogging application that enhances the blogging from just image and text to audio-video and text content along with "Interactive blogging"
The present invention relates to a method which helps users to create and participate in Video Blogging by inserting comments in the form of text, audio & audio-video instantaneously at suitable location while watching original blog in Real Time without modifying it. The comments inserted by user become a separate descriptor file & user needs to upload only descriptor file & not the video
& blog. Every comment uploaded by any user is stored as a separate descriptor file at the server. If there are multiple comments available on one blog then user can select specific comments as per his wish for downloading & watching.
The present invention further relates to a method wherein user can watch original blog along with the comments inserted by other users at various locations i.e comments inserted with respect to the content / time of insertion. The blog along with the comments can be viewed online or can be downloaded/streamed & viewed. The user can add or see comments interleaved with respect to the content / time of insertion, or with traditional back to back addition.
Accordingly the invention explains a method of interactive blogging comprising the steps of:
creating and inserting comments in the form of text, audio & audio-video at various location in real time without modifying the blog;
uploading and storing comments as descriptor file at a server; and
acquiring and playing original blog by a user along with the comments inserted by other users at various locations; where if there are multiple comments available on one blog, the user select specific comments accordingly.
The present invention further relates to a method wherein original blog & comments inserted by various users with respect to the content / time of insertion are separate entities therefore user has lot of flexibility in uploading, selecting comments, downloading/streaming, sorting & viewing as per his convenience/wish.
These and other objects, features and advantages of the present invention will become more apparent from the ensuing detailed description of the invention taken in conjunction with the accompanying drawings.

The preferred embodiments of the present invention will now be explained with reference to the accompanying drawings. It should be understood however that the disclosed embodiments are merely exemplary of the invention, which may be embodied in various forms. The present invention is a new use case/method for Video Blogging application that enhances the blogging from just image and text to audio-video and text content along with "Interactive AV blogging". User can create a video blog using a device having camera & recording facility. He can also use existing or pre¬recorded file as a video blog.
User can upload video blog to server or video blogging site using conventional/ traditional methods like as an e-mail attachment, ftp or the method recommended
^ by service provider or video blog site. The method suggested here assumes that some means are available to the user to upload video blog to server, video blog site using the device available with the user like mobile phone, feature phone, smart phone, PC or any handheld device. Once uploaded, accessing the same video blog by the owner or other users depends on the terms & conditions offered by host server/video blog site or service provider. The method suggested here assumes that there are some means available to access the uploaded video blog either by owner, some users or group of people which are allowed by owner (limited access) or video blog sites or service provider or can be accessed by anybody (global access). Owner of the video blog may share his blog to some people by entering their (other users) mobile numbers or user ids & may invite them to watch & participate in the video blog. The invited people might get some notification of the invitation through e-mail, sms or mms. If video blog is shared for all, then notification is not necessary & anybody can access uploaded video. People having access can watch the video blog online using web player. If any user wants to add his comment on video blog he needs to download the blog or should have a streaming client (player that can play data streamed by server). The method suggested here assumes that some means are available to the user to download/stream video blog from server, video blog site using the device available with the user like mobile phone, feature phone, smart phone, PC or any handheld device.
With the existing methods user can add only text comments. But with the help of
^r new invention (l-VB) proposed here, user can insert his comment in the form of text, audio or audio video at any point of time while playing video blog. User selects the video blog of his choice downloaded by him or selects video blog to be streamed from server & starts playing the video file.
The application shows two windows on LCD one playing video blog (W1) & another is empty (W2). Whenever he wants to insert the comment in the video blog he should select "Insert" option. Whenever he selects insert option, application pauses playback of video blog playing in one window (W1). User needs to select type of comment like text, audio or Audio-video. If user selects "text" option then other window (W2) shows text editor & allows user to input text character to add text comment. If user selects "Audio" option then application allows user to record his comment as audio clip. If user selects "Audio-Video" option then other window (W2) shows camera preview & user can record audio- video comment. User can resume playback of video blog from the same position in window W1 by using option "resume" after recording his comment. User can repeat above procedure of "insert" & "resume" while watching video blog to insert his comments in the form of text, audio or audio -video at various points of time.
The inserted comment does not affect original video blog. The inserted comment becomes a separate descriptor file. Depending on the option selected by the user each insertion becomes a separate descriptor file or all insertions during a session can be merged into one descriptor file.
The descriptor file has two major parts i.e. data & metadata. The purpose of metadata in descriptor file is to allow user to define the type of comment (text, audio or audio-video) & remember/record the information like for which file (downloaded/streamed video blog or for a comment on video blog) it is inserted, details of the video blog like date/time, at which point of time the comment is inserted while playing, video frame number, audio frame number of the video blog, time of insertion & offset. This information helps to co-relate or associate the descriptor file with the correct video blog or comment on video blog & provides the information about where exactly the comment is inserted. Along with the above information it stores the data (text, audio, video) inserted by the user in compressed /encoded form.
The above information helps application to find relation between the descriptor file & its associated video blog. It also gives perfect information of when a particular comment in descriptor file is inserted & type of comment. Using the above information it is possible to play original video blog along with the comments inserted at various point of time at correct time if original blog & descriptor file is available.
In descriptor file the text information is stored in ASCII form whereas audio, video information is stored in the format as used in original video blog. For example if original blog is recorded in H.263 Video & AAC format then inserted AV comment
^ is also stored in H.263 & AAC format. In order to support this functionality application in the beginning checks the codec type of video blog & initializes the same for inserting audio-video comments.
The present Invention proposes two formats of the descriptor file, one is proprietary (new/non-standard) & other is 3gp/mp4 format. In case of 3gp/ mp4 format any audio or audio-video comment inserted by user is recorded as per the 3gp/mp4 specification. But all other information like metadata & text comment is written as user defined atom. Due to this the descriptor file created by application can be played in any 3gp/mp4 compliant player. But as complete metadata is in user defined atom of the 3gp/mp4 descriptor file all players other than l-VB application can not co-relate descriptor file with its associated video blog & can not play comments at right time.
The present invention also suggests a proprietary way of writing metadata & data as a descriptor file. In this case l-VB records metadata & data required to associate descriptor file with a specific video blog along with the timing information. The disadvantage of this method is that only l-VB application can play & co-relate descriptor file with its associated video blog & can play comments at right time. There are two ways using which user can insert comments.User can download the blog & can play the blog in l-VB application for inserting comments or he can insert comments while playing streamed blog in I- VB applicationon. Thus after downloading video blog or while streaming the
original blog user can play it in l-VB application & can insert comments in the form of text, audio or audio -video file. The inserted comment does not become part of original blog. The above method helps user to participate in Video Blogging by inserting comments in the form of text, audio & audio-video instantaneously at suitable location while watching original blog in Real Time without modifying it. The comments inserted by user become a separate descriptor file. User needs to upload only descriptor file & not the video blog. Multiple users at a time can download/stream the blog & can participate in video blogging by creating descriptors files & then only uploading descriptor file to server & not the video blog. If there are already such descriptor files created & uploaded by some users & then new user can see original blog & various separate descriptor file on the server. Server shows listing of original blog, comments added & comment to comment added by the user as descriptor files. It is possible to play original video blog & selected descriptor files on line or through web-player if server runs l-VB application. But user can only watch video blog & inserted comments with respect to time & can not insert any comment through web -player.
New user can select the descriptor files of his choice & can download along with original blog. New user opens original blog along with descriptor file/files of his choice. If user selects play option then application starts playing original blog. Application shows two windows on LCD i.e. W1 & W2 & plays original blog in W1. Application checks the metadata in various descriptor files & keeps the track of the type of comment & when it was inserted in original blog. Based on the information available whenever it finds any comment needs to be played at particular instant it pauses the playback of original video blog in W1 & starts playing comment in W2. If the comment is text then it shows text comment in W2. For audio it plays audio comment whereas for audio video comment it plays comment in W2. It resumes playback when user selects resume or when the comment ends. Using the above method it is possible to see/watch all the comments inserted at various point of time during a playback of original video.
The application also provides a quick & easy access of the comments inserted in the blog & directly go to the selected comment & its associated video point in original video blog. User can select original blog along with descriptor files of his choice & then type of comments like text, audio or audio-video. Application provides a listing of all text or audio or audio-video comments along with video thumbnail, which is starting point of associated video in video blog. User can select a comment & can directly jump to that comment & its associated video. The same user if interested can insert a new comment after watching other comments & can upload to server.
The application also allows user to convert descriptor file to a new video blog. In this case it deletes all references to the original video blog. The owner of the video blog can update original video blog by merging some or all descriptor files to the original video blog & can again upload. For example user can merge 3
descriptor files with original blog out of 6 available on server. He needs to delete descriptor files merged with original file & need to update remaining to upload on to the server. The application also provides an option of playing all comments back-to back.
It is also possible to play/stream few or all the comments selected by user with respect to the time along with original blog at server if server uses l-V blog application.
User can watch original blog along with the comments inserted by other users i.e comments inserted with respect to. the content / time of insertion using streaming player as server streams down the data using the technique described above to play the original blog & comments using l-VB application.
