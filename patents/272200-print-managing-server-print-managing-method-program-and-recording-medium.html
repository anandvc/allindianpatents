Title of Invention

"PRINT MANAGING SERVER, PRINT MANAGING METHOD, PROGRAM, AND RECORDING MEDIUM"
Abstract A print managing server is connected to a plurality of printers through a network and stores print data which can be printed by the printer in combination with user information. The print managing server has control means for controlling execution of a command including the user information which is received from the printer. The control means controls in such a manner that when the command is received, if a printing process corresponding to the user information included in the command is being executed for the other printer, the printing process which is being executed is stopped for the other printer and a process corresponding to the command is executed.

TECHNICAL FIELD
The invention relates to control of a print managing server which is connected to a plurality of printers through a network and stores print data which can be printed by the printer in combination with user information.
BACKGROUND ART
Hitherto, a print system of what is called "pull print (accumulation print) " in which only the authenticated user makes a print request to print data which has temporarily been accumulated onto a server from a printing apparatus, thereby enabling the print data to be output from the printing apparatus has been proposed. Thus, the print data corresponding to the authenticated user can be output from a desired printing apparatus.
For example, in Japanese Patent Application Laid-Open No. 2006-99714, there has been disclosed a print system in which an IC card authenticating process is executed by a multi-function apparatus, print data of the authenticated user is specified by a print managing server, and a print command is transmitted to a print server in which the specified print data has been stored, thereby enabling the

print data of the authenticated user to be printed by a desired printing apparatus.
However, in the case of using the foregoing prior art by using authenticating means using, for example, contactless type IC card, once an authentication is made, a subsequent process (printing) is executed. Therefore, the IC card is not necessary until the process of a whole sheet responsive to a j ob to which an execution of the printing has been instructed is completed (or, the IC card is not necessary at all for the subsequent process) , so that another printing can be executed by another printer although the printing is being executed (particularly, when a large quantity of printing is being executed, since it takes a long time until a sheet discharge is long, such a situation is liable to occur).
That is, in the related art, there is such a problem that since one user can simultaneously execute the printing by a plurality of printers^ a state of a low security occurs.
DISCLOSURE OF THE INVENTION
An object of the present invention is to provide printers with an increased security by allowing only the printer which has sent user identification information to the print managing server to execute printing.
The invention is directed to a print managing server which is connected to a plurality of printers through a network and stores print data in combination with user

identification information. A reception means is provided to receive user identification information from a first one of the plurality of printers. A data transmission means is provided to transmit print data corresponding to the user identification information received by the reception means to the first printer. A control means is provided to control the data transmission means to stop the transmission of the print data to the first printer in accordance with reception of user identification information identical to the user identification information corresponding to the print data from a second one of the plurality of printers, wherein the control means controls the data transmission means to transmit to the second printer the print data which has not been printed due to the stop of the data transmission.
According to the present invention, the print managing server allows only the printer which has sent user identification information to the print managing server to execute printing, thereby providing printers with an increased security.
