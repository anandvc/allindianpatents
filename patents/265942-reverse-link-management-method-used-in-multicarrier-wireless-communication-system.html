Title of Invention

REVERSE LINK MANAGEMENT METHOD USED IN MULTICARRIER WIRELESS COMMUNICATION SYSTEM
Abstract A reverse link management method in a multicarrier wireless communication system, includes: the access terminal AT sends a Reverse Link Management Message to the access network AN, said Reverse Link Management Message carries information of the reverse link that the AT dropped; AN sends the Traffic Channel Assignment Message to AT, AN assigns a new reverse carrier link according to the received the Reverse Link Management Message; AT establishes the reverse carrier link (reverse link) based on said Traffic Channel Assignment Message; AT transmits data on the new reverse carrier link. In the invention AT could actively report the number of its containing reverse links to AN; also inform AN of the information of the reverse link it has dropped; at the same time, it can keep the backward compatibility. In addition, the Reverse Link Management Message length is short, and spares small air interface resource; therefore it can enhance the availability of air interface wireless resource.

Field of the Invention
[0001] The present invention relates to the field of wireless communication technology, and in particular to a management method and an apparatus for reverse link in a multicarrier wireless communication system.
Background of the Invention
[0002] Single-carrier transmission is adopted in the existing CDMA2000 IxEvolution-Data Optimized (IxEV-DO) technology. In a IxEV-DO system, an Access Terminal (AT) only uses one reverse carrier link to transmit information.
[0003] A higher data transmission rate is required for the development of data services. The Third Generation Partnership Project 2 (3GPP2) has proposed a new air interface evolution plan. In the first phase of the plan, a great increase in air interface data rate is enabled through Multiple Carrier EV-DO (NxEV-DO). It is required that the forward data rate is no lower than Nx3.1Mbps and the reverse data rate is no lower than NX l.SMbps, in which N is the number of carriers used by an Access Network (AN) or the AT. Further, it is required that the forward and reverse carriers be assigned dynamically and independently through the cooperation of the AN and the AT. The number of forward carriers and the number of reverse carriers are not required to be symmetric.
[0004] In an NxEV-DO system, an AT should be able to request an AN for the number of reverse carrier links the AT can support according to the AT's current power amplification state or other factors, and also be able to drop one or more reverse carrier links according to its power amplification state or other factors and report this to the AN.
[0005] Figure 1 is a diagram illustrating the structure of a IxEV-DO system, where the direction of a forward carrier link is defined from the AN to the AT and the direction of a reverse carrier link is defined from the AT to the AN. The components of the IxEV-DO system are respectively described as follows:
[0006] Access Network (AN) is a network equipment configured to provide data connectivity between a packet switched data network (typically the Internet) and the ATs, and the AN is equivalent to a base station in a CDMA2000 system.
[0007] Packet Control Function unit (PCF unit) mainly provides between a wireless link and a wire link functions such as mapping, call control, mobility management, session management, data buffering and forwarding, and accounting information collection.

[0008] Packet Data Serving Node (PDSN) mainly provides functions such as link management, mobility management, address distribution, accounting information collection, data routing and tunnel encapsulation, and also supports the function of foreign agency for Mobile IP.
[0009] Accounting, Authentication, Authorization server (AAA server) mainly provides functions such as identity authentication and address distribution.
[0010] Mobile Switching Center (MSC) is a circuit domain voice switching center through which the circuit domain voice service is introduced.
[0011] In the existing IxEV-DO system, when performing a configuration negotiation or establishing a traffic channel, the AT needs to establish a connection. There are mainly the following situations:
[0012] The AT initiates a connection request on its own initiative;
[0013] The AN sends a paging message, and the AT initiates a connection request when receiving the paging message;
[0014] The AT initiates a connection request when an initial establishment of a session or a configuration negotiation is required.
[0015] As shown in Figure 2, the message flow in the case that the AT initiates a connection request on its own initiative includes the following processes.
[0016] SOI: The AT sends a Connection Request.
[0017] S02: The AT sends a Route Update message to indicate the wireless circumstance.
[0018] 803: The AN sends a Media Access Control (MAC) layer Acknowledgement message to confirm that the Route Update message is received.
[0019] S04: The AN sends a Traffic Channel Assignment message.
[0020] SOS: The AT establishes a Reverse Traffic Channel (RTC) and sends a pilot signal and a Data Rate Control (DRC) signal..
[0021] S06: The AN sends an RTC ACK message to confirm that the RTC is received.
[0022] S07: The AT sends a Traffic Channel Complete message on the reverse traffic channel.
[0023] S08: The AT starts to exchange data with the AN. [0024] It can be seen from the prior art:

[0025] The AT is unable to request the AN for a reverse carrier link;
[0026] The AN does not know the number of reverse carrier links desired by the AT;
[0027] Only one reverse carrier link can be assigned to the AT, and the peak rate of the reverse carrier link is just 1 .SMbps;
[0028] The forward and reverse data rates can not meet the requirement of the future service.
Summary of the Invention
(0029] An embodiment of the invention provides a reverse link management method in a multicarrier wireless communication system, so that the AT can flexibly adjust the reverse carrier links according to its capability of supporting the reverse carrier links.
[0030] A reverse link management method used in a multicarrier wireless communication system according to an embodiment of the invention includes:
[0031] sending, by an Access Terminal, a Reverse Link Management message to an Access Network (AN), the Reverse Link Management message carrying information of one or more reverse carrier links that the Access Terminal has dropped.
[0032] Preferably, the Access Network assigns one or more reverse carrier links according to the received Reverse Link Management message and sends a Traffic Channel Assignment message to the Access Terminal;
[0033] the Access Terminal establishes the one or more reverse carrier links according to the Traffic Channel Assignment message.
[0034] Preferably, the Reverse Link Management message is transmitted on a reverse access channel or a reverse traffic channel.
[0035] Preferably, the Reverse Link Management message carries information of the number of reverse carrier links that the Access Terminal currently can support.
[0036] Preferably, the Reverse Link Management message contains a message sequence number, a link dropped indication and reverse link index information.
[0037] Preferably, a correspondence relationship exists between the reverse link index information of the Reverse Link Management message and the one or more reverse carrier links assigned according to the Traffic Channel Assignment message.
[0038] Another reverse link management method according to an embodiment of the

invention includes the following processes:
[0039] sending, by an Access Terminal, a first Reverse Link Management message or a second Reverse Link Management message to an Access Network (AN), the first Reverse Link Management message carrying information of one or more reverse carrier links that the Access Terminal has dropped and the second Reverse Link Management message carrying information of the number of reverse carrier links that the Access Terminal currently can support.
[0040] Preferably, the reverse link management is further implemented through the following processes:
[0041] the Access Network sends a Traffic Channel Assignment message to the Access Terminal so as to assign one or more new reverse carrier links according to the received first Reverse Link Management message or the received second Reverse Link Management message;
|0042] the Access Terminal establishes the one or more reverse carrier links according to the Traffic Channel Assignment message.
[0043] Preferably, the first Reverse Link Management message or the second Reverse Link Management messeige is transmitted on a reverse access channel or in an existing reverse traffic channel.
(0044] Preferably, the first Reverse Link Management message contains a message sequence number, a link dropped indication and reverse link index information.
[0045] Preferably, a correspondence relationship exists between the reverse link index information of the first Reverse Link Management message and the one or more new reverse carrier links assigned according to the Traffic Channel Assignment message.
[0046] An Access Terminal is further provided according to an embodiment of the invention, which includes a terminal entity, a link control unit and a message transmission unit, wherein
[0047] the message transmission unit sends a Reverse Link Management message to an Access Network (AN), according to a control signal sent by the link control unit;
[0048] the link control unit receives a Traffic Channel Assignment message sent by the Access Network, and controls the Access Terminal to establish one or more reverse carrier links according to the Traffic Channel Assignment message.
[0049] To sum up, in the technique solution of an embodiment of the invention, the Access Terminal can request the Access Network for the desired number of reverse carrier links and

also can report to the Access Network the information of one or more reverse carrier links the Access Terminal has dropped.
[0050] The Reverse Link Management message does not make any modification to the existing messages. Therefore, the backward compatibility can be kept.
[0051] The Reverse Link Management message has a short length and occupies less air interface wireless resource.
[0052] The Access Terminal may request or drop one or more reverse carrier links dynamically and flexibly according to need.
[0053] The Access Terminal sends a Reverse Link Management message to the Access Network on an access channel to request the number of reverse carrier links supported by the Access Terminal according to its current situation (such as the power amplification state and the data quantity in a buffer). Therefore, the Access Terminal can make fully use of the power amplification resource and increase the utilization rate of the air interface wireless resource.

Detailed Description of the Embodiments
[0059] In order to make the principle, characteristic and advantage of the invention clearer, the specific embodiments of the invention will be described by taking an NxEV-DO system as an example with reference to the attached drawings.

[0060] The embodiments of the present invention can be applied to the NxEV-DO system and also are compatible with the IxEV-DO system. The improvement is made on the C.S0024-A-V1.0 protocol.
[0061] Figure 3 is a schematic diagram illustrating the structure of an AT of an embodiment of the invention. It can be seen from Figure 3, two functional units are added to the existing terminal: a reverse carrier link control unit and a message transmission unit.
[0062] By using the two functional units, the AT may exchange reverse carrier link messages with the AN and control to establish one or more reverse carrier links.
[0063] Specifically, the message transmission unit sends a Reverse Link Management message to the AN according to a control signal sent by the reverse carrier link control unit.
[0064] The reverse carrier link control unit receives a Traffic Channel Assignment message sent by the AN, and establishes one or more reverse carrier links of the AT according to the Traffic Channel Assignment message.
[0065] A Reverse Link Management message is set in an embodiment of the invention. The message can indicate the number of reverse carrier links the AT can support currently and also can indicate the information of one or more reverse carrier links the AT has dropped. The Reverse Link Management message is sent by the AT to the AN.

[0067] Firstly, the above fields are respectively explained as follows:
[0068] Message ID refers to a message identification. Message IDs are used to distinguish different messages so that each message corresponds to a unique Message ID.

|0069] MessageSequence refers to a message sequence number. The AT sets this field to take the MessageSequence value of the latest Traffic Channel Assignment message it received.
|0070] LinkDroppedlndication refers to a link dropped indication. If this field is set to ' 1', the message indicates that the AT reports information of one or more reverse carrier links the AT has dropped to the AN. If this field is set to '0', the message indicates that the AT reports the number of reverse carrier links the AT can support currently to the AN.
|0071] Num of Reverse Links refers to the number of reverse links.
(0072] If the LinkDroppedlndication is "0", the field Num of Reverse Links indicates the number obtained by subtracting 1 from the number of reverse carrier links which the AT reports to the AN it can support. If the LinkDroppedlndication is "1", the field Num of Reverse Links indicates the number obtained by subtracting 1 from the number of reverse carrier links the AT has dropped.
[0073] ReverseLinklndex refers to a reverse link index. The field ReverseLinklndex is included in the message only when the LinkDroppedlndication field is "1". Otherwise, the field ReverseLinklndex is not included in the message. The field ReverseLinklndex is consistent with the sequence of the reverse carrier frequency points indicated by the Traffic Channel Assignment (TCA) message which has the same MessageSequence field as that of the message including the field ReverseLinklndex. For example, if the field ReverseLinklndex is '0000', it indicates a reverse carrier link with the lowest frequency point in the corresponding TCA message. If the field ReverseLinklndex is '0001', it indicates a reverse carrier link with the frequency point immediately next to the lowest frequency point in the TCA message. The rest cases may be deduced by analogy.
[0074] Reserved refers to this field is reserved. The field Reserved is set to be all '0' by the AT, and the number of '0' makes the length of the message be integral times of 8 bits.

[0080] Embodiment 1: An AT in an idle state initiates a Reverse Link Management message on its own initiative.
[0081] Figure 4 is a flowchart illustrating a method of initiating a Reverse Link Management message by an idle AT on its own initiative according to an embodiment of the invention. As shown, the method includes the following processes:
[0082] Sll: The AT sends a Reverse Link Management message on an access channel to the AN, by means of which the AT requests the number of reverse carrier links the AT can support according to the AT's current condition (e.g., power amplification state, data quantity in the buffer).
[0083] S12: The AN sends a Traffic Channel Assignment message according to the received Reverse Link Management message and the Traffic Channel Assignment message includes specific information of the forward and reverse channels.
[0084] In the NxEV-DO system, the Traffic Channel Assignment message sent by the AN needs to be expanded on the basis of C.S0024-A-V1.0. The Traffic Channel Assignment message includes information of relevant channels.
[0085] The forward channel information includes carrier frequencies of one or more forward carrier links, sector information for transmission of the forward channel, MAC index information assigned to the AT for identifying data packets transmitted to the AT and soon.
[0086] The reverse channel information includes carrier frequencies of one or more reverse carrier links, MAC index information of a power control channel (a forward channel used for controlling the transmission power of the reverse channel), forward channel index information for transmitting the power control channel and so on.
[0087] SI3: The AT establishes one or more newly assigned reverse carrier links according to the contents contained in the Traffic Channel Assignment message and sends a pilot signal

and a DRC signal to the AN.
[0088] The reverse pilot channel is mainly provided to help the AN to hunt a reverse channel and to implement coherent demodulation.
[0089] The DRC channel is a data rate control channel transmitted by the AT for notifying the AN of the forward traffic channel rate requested by the AT and the service sector selected by the AT.
[0090] SI4: The AN sends an RTC ACK message to acknowledge receipt of the reverse traffic channel.
[0091] SI5: The AT sends a Traffic Channel Complete message on the reverse traffic channel and shuts down the timer.
[0092] The AT starts the timer when sending the connection request. If the connection can be established within the time period determined by the timer, the timer is shut down. Otherwise, the time period determined by the timer expires and the AT is changed to another state.
[0093] SI6: The AT and the AN start to exchange data on both the prior and newly assigned traffic channels.
[0094] Embodiment 2: An AT in an active state initiates a Reverse Link Management message.
[0095] Figure 5 is a specific flowchart illustrating a method of initiating a Reverse Link Management message by an AT in the service state according to another embodiment of the invention. As shown, the method includes the following processes:
[0096] S21: A traffic channel is established. The process for establishing the traffic channel is the same as the information exchange process used by the AT to initiate a connection request as illustrated in Figure 2 and therefore the description is omitted here.
[0097] S22: After the traffic channel is established, the AT determines whether the reverse carrier links it can support have changed or whether it has dropped one or more reverse carrier links.
[0098] S23: The AT sends a Reverse Link Management message to the AN to request the number of reverse carrier links the AT can support currently or to report information of the one or more reverse carrier links the AT has dropped.
[0099] S24: The AN sends a Traffic Channel Assignment message to assign the forward and reverse carrier links. When sending the Traffic Channel Assignment message, the AN

assigns with its best effort the number of reverse carrier links which can meet the number requested by the AT according to the received Reverse Link Management message.
[0100] S25: The AT establishes one or more reverse carrier links and sends a pilot signal and a DRC signal to the AN.
|0101] S26: The AN sends an RTC Ack message to acknowledge receipt of the reverse traffic channel.
[0102] S27: The AT sends a Traffic Channel Complete message on the reverse traffic channel.
[0103] In embodiments of the present invention, the AT can send a Reverse Link Management message to the AN on its own initiative to request the number of reverse carrier links the AT can support or to report information of one or more reverse carrier links the AT has dropped. The Reverse Link Management message does not make any modification to the existing messages. Therefore, the backward compatibility can be kept. The embodiments of the invention enable the AT to request or drop one or more reverse carrier links dynamically and flexibly according to need, and the AT can make fully use of the power amplification resource and increase the utilization rate of the air interface wireless resource.
[0104] In an embodiment of the invention, the function described above can also be achieved by using two separate messages. One message is used by the AT to inform the AN of the number of reverse carrier links the AT can support currently. The other message is used by the AT to report the information of one or more reverse carrier links the AT has dropped to the AN. The message exchange flow is in conformity with that shown in Figure 4 or Figure 5.
