Title of Invention

METHOD FOR MANAGING ACTIVITIES AND PROVIDING AN ALERT IN GLOBAL POSITIONING SYSTEM (GPS) ENVIRONMENT
Abstract The present invention relates in general to the Mobile Telecommunication, Wireless Network. More particularly the present invention relates to a method for managing activities and providing an alert in GPS environment. The invention explains a method for managing activities and providing an alert in GPS enabled mobile phones when entering a specific location comprising the steps of: downloading the GPS maps of the specific locations by the GPS enabled mobile phone; storing the GPS co-ordinates of the said specific locations and assigning a name to each such location stored in the mobile phone; scheduling the appointments by selecting the location along with a radius value by the user; and alerting the user with the list of things to be completed at that location once the user enters the specific location.
Full Text FIELD OF THE INVENTION
The present invention relates in general to the Mobile Telecommunication and Wireless Networks. More particularly, the present invention relates to a method for managing activities and providing an alert in GPS environment.
DESCRIPTION OF RELATED ART
Currently the mobile phone user schedules the appointments in the scheduler by assigning date/time (when the activity needs to be completed) to the respective activity. The user is alerted for the activity at the specified time. In such a solution there is no provision where in the user can specify the appointments that she/he would like to complete once they reach a specific place (e. g. home/office/market/friends place).
Following are the steps for the related art as it exists:
1. The user selects the scheduler from the user interface menu options.
2. The user inputs the activities along with the date/time when the reminder needs to be provided to the user.
3. At the specified date/time of a certain activity the software in the mobile phone alerts the user with the activity.

A back ground process keeps on monitoring if the user has to be alerted for a specific activity at a specific time. If the user needs to be alerted then at the specified time an alert is sent to the user through the user interface along with the activity to be completed.
The existing art does not provide an option of co-relating a scheduled activity with the place where it has to be completed. The current art just provides means to co-relate the time with the activity, this leaves the user with making it sure that at the designated time she/he is at the place where the scheduled activity has to be completed.
Additionally if the user is at a location prior to the scheduled time, the user is not alerted for the activity to be completed at that location as the activity is linked to the time and not the location
SUMMARY OF THE INVENTION
The object of the invention is to provide a solution along with the scheduler to link the activities that need to be completed at a specific place with date/time being optional.

The proposed invention is a solution in GPS enabled mobile phone for linking the activities with the location where they need to be completed and a method for alerting the user with the scheduled activities at the specified place.
Accordingly, this invention explains a method for managing activities and providing an alert in GPS enabled mobile phones when entering a specific location comprising the steps of:
(a) downloading the GPS maps of the specific locations by the GPS enabled mobile phone;
(b) storing the GPS co-ordinates of the said specific locations and assigning a name to each such location stored in the mobile phone;
(c) scheduling the appointments by selecting the location along with a radius value by the user;
(d) alerting the user with the list of things to be completed at that location once the user enters the specific location.
The said radius value is the radius of the region from the GPS co-ordinate of the location, entering into which the user wants to be alerted with the activities associated with that location. The step of scheduling the appointments includes preparing a list of things to be completed at the specified location. The step of scheduling the appointments includes providing a time line by the user within which each task needs to be completed. Scheduling the appointments involves the user prepares a list for each location which contains the activities to be completed at that location and optionally the time line associated with each

activity. A user entered specific location is identified by the radius from the GPS Co-ordinate of the location. The user provides the periodicity with which the GPS co-ordinates needs to be checked with the specified location. Also the user provides a value to indicate the number of times the user needs to be alerted with the list of activities upon entering a specified location.
These and other objects, features and advantages of this invention will become more apparent from the ensuing detailed description of the invention taken in conjunction with the accompanying drawings.

Figure 3 depicts the flow chart of how the provisioning of the activities based on the location cans be done and also how the user is alerted with the list of activities once she/he enters the region as specified by the radius associated with each location.
Figure 2 explains the working of the invention in a flowchart.
1. The user first configures and stores the list of activities to be done at each location in the GPS enabled mobile.
2. The user also provides the radius value with each location co-ordinates.

3. The GPS enabled mobile continuously or periodically monitors the current GPS location with the locations specified in the scheduler.
4. Then a check is made to find out if the distance between the current and the specified location is less than the radius.
5. If the said value is not less than the radius then monitoring is resumed with the interval specified by the user.
6. If the said value is less than the radius then a check is made for the number of reminders.
7. If the maximum limit for the reminder has not reached then the user is alerted with the associated list of the specified location.
The invention proposes the following to link the activities with the place where they need to be completed:
1. The GPS enabled mobile phone user downloads the GPS map of the locations where she/he visits frequently. This map can be downloaded from various software that provides GPS maps.

2. The mobile user stores the GPS co-ordinates of these locations and optionally assigns a name to each such location stored in the mobile phone.
3. When scheduling the appointments the user selects the location along with a radius value (this value is the radius of the region from the GPS coordinate of the location, entering into which the user wants to be alerted with the activities associated with that location) and prepares a list of things to be completed at the specified location. If desired the user can also provide a time line within which each task needs to be completed.
4. In this way the user for each location prepares a list which contains the activities to be completed at that location and (optional) the time line associated with each activity. This would enable the user to provide either just Location OR just timeline OR both location and timeline.
5. Once the user enters the regions (identified by the radius from the GPS Co-ordinate of the location) the user is alerted with the list of things to be completed at that location.
6. The user can also provide the periodicity with which the current GPS coordinates needs to checked with the specified location in order to determine if the user has entered one of the regions where the alert has to be provided.

7. Also the user can provide a value to indicate the number of times she/he needs to be alerted with the list of activities upon entering a specified location.
