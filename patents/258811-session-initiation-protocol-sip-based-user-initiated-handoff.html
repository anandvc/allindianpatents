Title of Invention

SESSION INITIATION PROTOCOL (SIP) BASED USER INITIATED HANDOFF
Abstract Method and apparatus for performing wireless handoff of a real time media session between two non-compatible devices such as PC 12 and UE 32, as shown at S. A WLAN 14 triggers a handoff and the WLAN 14 communicates with an IP network 18, shown at S2 The IP network 18 provides the two-way communication link between the PC 12 and service server 22, shown at S3. Both the user PC 12 and UE 32 establish a connection with the network 23 at S4-S6. The user then decides to trigger a handoff from PC 12 to the UE 32 at S7 and communicates this handoff request to the WLAN network 14, which transmits an SIP message through IP network 18 to service server 22. Service server 22 transmits an SIP 200 (OK) message to acknowledge the handoff (HO) message.
Full Text FIELD OF INVENTION
[0003] The present invention relates to performing a handoff between
two devices which employ different technologies and which are connected to an internet protocol (IP) network by way of two different systems. More particularly, the present invention is related to a method and apparatus for performing a handoff employing SIP protocol conducted during real time sessions between such different devices, and either such different networks

or a common network.
[0004] BACKGROUND
[0005] Session initiation protocol (SIP) protocol is employed to initiate
and to modify a! multimedia session over the internet protocol (IP) network. For example, SIP is used in one of a universal mobile telecommunications system (UMTS) network to establish a multimedia session between two users wherein at least one of the users operates on the UMTS system. However, SIP protocol has not heretofore been utilized as a mechanism to perform handoff between different devices operating in different systems.
[0006] SUMMARY
[0007] The present invention is characterized by method and apparatus
for providing a unique solution to the handoff problem including negotiation of new media codecs (coders/decoders) for utilization in cases where the two devices involved in the handoff are incompatible. Although mobile IP (MIP) is presently utilized to perform handoff procedures, this technique lacks a trigger mechanism to initiate handoff between two different devices which communicate with two different networks or one common network. In addition, MIP dbes not address the issue of compatibilities between media types, codecs and supported bit rate.

1. A mobile user equipment (UE) comprising:
a transmitter configured to:
communicate via a first network; and
transmit a first Session Initiation Protocol (SIP) message to the first network, wherein the first SIP message indicates a request for a handover of a communication session from another user equipment (UE).
2. The user equipment (UE) of claim 1, wherein the first SIP message includes a SIP invite message or a SIP handoff message.
3. The user equipment (UE) of claim 1, wherein the first SIP message is configured to indicate the other user equipment (UE) using an Internet Protocol (IP) address, an e-mail address, or a telephone number.
4. The user equipment (UE) of claim 1, wherein the transmitter is configured to transmit the first SIP message to a service server.
5. The user equipment (UE) of claim 1, wherein the first SIP message is configured to indicate the communication session using a media type, an IP address, a bit rate, or a codec.
6. The user equipment (UE) of claim 1, further comprising a receiver configured to receive a second SIP message that indicates an acknowledgment of the handover in response to the first SIP message.
7. The user equipment (UE) of claim 1, wherein the other user equipment (UE) is configured to communicate with the first network.
8. The user equipment (UE) of claim 1, wherein the other user equipment (UE) is configured to communicate with a second network having a different architecture than the first network.
9. A method of wireless communication, the method comprising:
communicating via a first network; and

transmitting a first Session Initiation Protocol (SIP) message from a first mobile user equipment (user equipment (UE)) to the first network, wherein the first SIP message indicates a request for a handover of a communication session from a second user equipment (UE) to the first user equipment (UE).
10. The method of claim 9, wherein the first SIP message indicates the second user equipment (UE) by an Internet Protocol (IP) address, an e-mail address, or a telephone number.
11. The method of claim 9, wherein the first SIP message indicates the communication session by a media type, an IP address, a bit rate, or a codec.
12. The method of claim 9, further comprising receiving a second SIP message that indicates an acknowledgment of the handover in response to the first SIP message.
13. The method of claim 9, wherein the second user equipment (UE) is in communication with the first network.
14. The method of claim 9, wherein the second user equipment (UE) is in communication with a second network having a different architecture than the first network.
