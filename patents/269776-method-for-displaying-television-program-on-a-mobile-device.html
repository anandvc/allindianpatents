Title of Invention

METHOD FOR DISPLAYING TELEVISION PROGRAM ON A MOBILE DEVICE
Abstract The present invention relates to a method for displaying TV program on mobile devices comprising the steps of detecting the mobile device by the TV using a detection mechanism; connecting the mobile device with the TV using a bearer mechanism; sending the channel details by the mobile device to the TV; and streaming the current channel contents to the mobile device based on the channel details. According to the invention, when a user is in the vicinity of the TV, the user's mobile device is detected by the TV, using a detection mechanism such as Radio Frequency Identification (RFID). Once the mobile device is detected, a short range wireless bearer such as-Bluetooth, Infrared, Wireless LAN or Ultra Wide Band (UWB) is triggered and the mobile device is connected to the TV. A wired bearer mechanism such as Universal Serial Bus (USB) can also be used for connecting the mobile device to the TV. Further, the mobile device sends 'user selected channel' details such as channel number, mode in which channel has to be viewed etc. to the TV. The TV can then stream the current channel contents to the mobile device (primary mode) or the TV can continue displaying the current channel contents on the TV screen and stream the contents of a different channel to the mobile device (secondary mode). Real time Transport Protocol (RTP) can be used to stream the contents of the channel from the TV to the mobile device.

Referring to Figure 1, it shows the block diagram of the method of TV display on mobile device. When a user is in the-vicinity of the TV, his mobile device will be detected by the TV automatically by using the available detection mechanism like RFID. Once the mobile device is detected, it will trigger the short range wireless bearer like Bluetooth, WI_AN, UWB or Infrared and connect the TV to the mobile .device. Bearer mechanism to connect to the TV can also be wired (USB or Serial).

The mobile device \N\\\ send its profile to the TV for intelligent filtering of channels. This profile could be used to identify and authenticate the user for viewing channels. For example, this authentication can be used to restrict kids from viewing adult channels.
The mobile device can be used to select a particular channel from TV using the connection bearer between TV and the mobile to view it on the mobile screen.
This will involve below steps:
a) Software application in the mobile device will send the "user selected channel" details to TV. These details could include the channel number and mode in which channel has to be viewed.
The protocol for sending this information will be defined.
The below modes are defined by this method:
Primary mode: In this mode, TV will stream current channel contents to mobile. It can also enter a stand by mode where it will switch off its screen if the user wants to view the channel contents only in mobile.
Secondary mode: In this mode, the TV will continue displaying the current channel contents in TV screen. TV will filter the new channel requested by mobile device and stream the video to mobile device.

b) The protocol for this streaming could be RTP. This method can also define
protocol for using Mobile [TV standards like DMB or DVB-H over short range
I
wireless medium.
c) This Video stream will be displayed on the mobile screen using the protocol defined by this method. The method will also define presentation mechanism'for richer user experience for viewing the TV contents on mobile screen.
Also the request for seeing the program on mobile device can also be made using an auxiliary device like remote control. For example, this could be used to alert a member in some other room of the house to view the channel contents in mobile.
A software framework in TV with a protocol for doing the following will be defined:
1. Get the Request from mobile device/auxiliary device
2. Authenticate the user request based on profile sent by the mobile device.
3. Process the request and get the requested channel contents. Here TV might require capability to tune two channel contents at same time.
4. Send the channel contents as a stream to mobile device.

5. A Master console for viewing all secondary devices connected to TV and channels being viewed on them.
A Protocol for packaging the video streams and sending to mobile device will be defined.
A software framework in mobile device with a protocol for doing the following will be defined:
1. Sending user profile to TV
2. User Interface for displaying details of channel available for viewing through mobile

3. Packaging and sending the user request along with user selected viewing ' mode to TV .
4. Receive Video streams from TV and display it in mobile screen

5. Presentation of the TV content ih mobile screen including controls for user available at this stage. E.g. channel controls, volume controls etc.
6. To handle interrupts/events in mobile device
