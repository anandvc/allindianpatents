Title of Invention

METHOD AND SYSTEM OF IMPLEMENTING CALL AUTHORITY BASED ON INTELLECTUAL MOBILE NETWORK
Abstract The present invention relates to a method of implementing call authority based on mobile intelligent network and the system thereof, and it performs distribution of numbers for service users and configuration of service data on service switching point devices and wireless-side devices of a mobile intelligent network system, which includes distribution of numbers for the service users, configuration of restricted position areas, and assignment of home units, etc.; and it loads a service logic software which realizes the intelligent network service at a service control point device of the intelligent network, when a service user or a non-service user makes a call service, the service logic software and the service switching point control the call authority of the service user or non-service user according to the preset parameters, therefore different users can have different call authorities when they are in different areas. The demands such as call security of the user in specific location can be achieved by the method of the present invention.

Technical Field
The present invention relates to an intelligent network of a mobile network, and
especially, relates to a method of implementing call authority of a mobile intelligent network
and the system thereof.
Background Art
The mobile network telephone, as a communication means, has currently developed
rapidly worldwide because of its characteristics that it is portable and can be use at any time
and any place, and the government departments of some countries bring this communication
means into their daily office work. When using the mobile telephone, some secret-guarding
departments such as the jail and detention center bring forward the following demands:
when the users of this department are inside the department (specific geographic location),
they can call their colleagues in the same department, but can not communicate with people
outside the department, so as to avoid the leak of sensitive messages to the outside fast;
when the users of this department are not inside the department, then they are permitted to
call and answer calls as a common user does; furthermore, when a user who is not a user of
this department enters into this department, the user is not permitted to call or answer calls.
These demands that the call authority is decided according to the location are still not solved
in the prior art.
Summary of the Invention
The technical problem that the present invention aims to solve is to provide a method of
implementing call authority based on a mobile intelligent network and the system thereof,
and the call authority is decided by the present location of a user terminal so that the
demands such as call security of the user at a specific location can be achieved.
In order to achieve the above objects, the present invention provides a method of
implementing call authority based on a mobile intelligent network, characterized in
comprising the steps of:
step 1: building a mobile intelligent network system, and the system including service
switching point devices, a service control point device and wireless-side devices of the mobile
network: a base station device and a base station control device;
step 2: completing configuration of service data on the service switching point devices
and wireless-side devices, including: planning the coverage area of the base station of a
specific department and deploying base station devices, so that the geographic area of the
specific department is just covered by these base stations; defining a restricted position area
in the service switching point devices of the mobile intelligent network, so that the restricted
position area covers the geographic area of the specific unit which needs to redistribute the

call authority; furthermore, a user belonging to the specific unit subscribing an intelligent
network service being referred to as a service user, and a user not subscribing the intelligent

step 3: the service switching point devices of the mobile intelligent network providing
management function of a call barring service area, for the non-service user, setting the
restricted position areas as a service use-barred area at the service switching point device of

The above method of implementing call authority based on a mobile intelligent network,

same specific department; and
when the non-service user or the service user is outside the restricted position area, they
are permitted to call freely.

Embodiments for Carrying Out the Present Invention
The following is a further detailed description of the implementation of the technical
solution in conjunction with the accompanying drawings, and the accompanying drawings
show a system based on a mobile intelligent network—PHS (Personal Handyphone System)
intelligent network, and the following description is made in conjunction with the PHS
network.
In the present invention, the concept of PA (Position Area) is defined, and a position area
is composed of some base station coverage areas, and a PA designates a geographic area
since the geographic location of the base station is fixed. Furthermore, when a user is using
the PHS telephone, the PHS telephone certainly will make a call or receive a call through a
base station, so the user and the base station are in the same area, if the base station is
designated to belong to a PA, then it can be confirmed that the user is in the PA. The present
invention designs that each PA is planned by the service provider, so as to achieve the
purpose of distinguishing the call authorities. The PA is in logic sense, which can be
illustrated by a numeric string of fixed length, such as 0250528700001234.
The system of the present invention adopts the manner of intelligent network device, as
illustrated in figure 1, which is mainly composed of the following parts: a service switching
point device 20 (Service Switching point, SSP), a service control point device 10 (Service
Control Point, SCP), and the wireless-side devices of the PHS intelligent network such as a
base station device 40 and a base station control device 30. In the present invention, the
4

SSP device has the function of managing PA, and the management of PAs just needs
operations on the SSP, and the SSP can use the CAP protocol in the No. 7 signaling to trigger
the intelligent network service, when triggering the service, it transmits the present PA
parameter of a user 50 (PHS telephone) to the service loaded on the SCP.
Besides planning PA, the service logic software as described in the present invention will
be loaded on the SCP, and numbers will be distributed, position areas will be configured, and
home departments (groups) will be assigned for the service users (PHS telephone). After
completing the configuration of the service data, when a service user makes a call or other
users call the service user, the service logic will control whether the service user can connect
the call according to the preset parameters.
The present invention demands that the configuration of the service data be completed in
the SSP and the radio network device firstly. These configurations of data comprise:
(1) planning the base station coverage area of a specific department (group), so that
t he geographic area of the specific department (group) is just covered by some
base stations; defining the PA areas in the SSP, and these PAs cover the areas
which need to redistribute the call authority, i.e., composed of the base stations
which cover the geographic area, and setting these areas as the "service
use-barred area" in the SSP, i.e., the restricted position area, then when a
common user enters into these areas, the common user can not make/answer
calls or receive/send short messages.
(2) As for the telephone of a user of the specific department (group), setting in the
SSP that the service (i.e., when these users enter into the area defined in step 1,
they can make or answer calls) can be used in the "service use-barred area" and
subscribing an intelligent network service (these users are referred to as service
user in the present i nvention) for these users.
The service logic software as described in the present invention needs to be loaded on the
SCP, and the number of the service user (PHS telephone), the home department (group) of
the service user, and the restricted position area (PA) of the service user need to be stored in
the service system of the SCP.
The service in the present invention has the characters as follows:
the service divides several PAs into a restricted position area by using the aforesaid PA.
When a non-service user enters into this area, the non-service user can not answer/make
calls under the control of the SSP; when the present service user makes or answers calls in
this area, only the call with the user of the same department (group) is permitted by the
service.
The PA data is transmitted to the service through the IDP (Initial Detection Point)
parameter at the beginning of the call, and the service provides the function of setting the
corresponding relationship between the PA and the restricted position area.

Figure 6 is a schematic view of call flow according to the present invention, which explains
the method for processing the call of a service user by the present invention:

the SSP;
step 602, the system obtaining the calling number, called number and the PA parameter

step 608, the system releasing the call, then the calls is over.
The above call flow just explains the call processing flow of the service user, for the
non-service user, its call authority is determined by the service switching point, and the call
processing flow is similar to above mentioned, and it will not be described again.
As illustrated in figure 2, in conjunction with figure 6, the explanation of the service user
as a calling user in the non-restricted position area is as follows:
1. a service user dialing numbers and making a call, and triggering the present

2. the system obtaining the calling number, the called number and PA parameter
f rom the service triggering data.
3. the system matching the restricted area according to the PA parameter,
determining that the user is not in the restricted area, connecting the call, the
calling user and the called user talking with each other, and then the call is over.
As illustrated in figure 3, in conjunction with figure 6, the following is the explanation of
the circumstance that a service user, as the calling user, dials the telephone of a user of the
present department in the restricted position area:
1. a service user dialing numbers and making a call, and triggering the present
service of the SCP through the subscription data in the SSP.
2. the system obtaining the calling number, the called number and PA parameter
from the service triggering data.
3. the system checking the service data according to the calling number and the
called number, determining that the calling user and the called user are the users
of the same department (group), connecting the call, the calling user and the
called user talking with each other, and then the call is over.
As illustrated in figure 4, in conjunction with figure 6, the following is the explanation of
the circumstance that a service user, as the calling user, dials the telephone of a user who
does not belong to the present unit in the restricted position area:
1. a service user dialing numbers and making a call, and triggering the present
service of the SCP through the subscription data in the SSP.

2. the system obtaining the calling number, the called number and PA parameter
from the service triggering data.
3. the system matching the restricted area according to the PA parameter,
determining that the user is not in the restricted area, checking the service data,
determining that the calling user and the called user are not users of the same
department (group), releasing the call, and then the call is over.
As illustrated in figure 5, the following is the explanation that the non-service user, as the
calling user, makes a call in the restricted position area:
1. a non-service user dialing numbers and making a call, and starting the call
processing flow in the SSP.
2. the SSP judging that the user is in the restricted position area and judging that the
user is not the subscribed service user according to the user data stored in the
present system.
3. the SSP releasing the call, and the call is over.
From the above embodiments, it can be seen that the call authorities of different users is
distinguished from each other through the method of setting the restricted area according to
the present invention, no matter for a non-service user or a service user, and no matter being
a calling user or a called user, the effect of decentralized authority restricting call can be
achieved, as a result, the effect that different users have different call authorities in different
areas can be achieved.
In the GSM network and the CDMA network, there is also a network protocol similar to the
above PHS, transmission of information such as the calling number, the called number and
the position parameter is provided in the protocol, so the method proposed by the present
invention fits these networks, too.

Industrial applicability
Compared with the prior art, the present invention can flexibly set specific areas for a
user, and the effect of restricting any call of a user or the effect of only being able to make
calls in a certain area by the user can be achieved by the method of call authority, and the
service provider can provide the present invention to departments such as jails and detention
centers, so that their demand of keeping secret in using the PHS telephone can be met.
